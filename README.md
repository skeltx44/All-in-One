# 🌱 All in One

> 진로 시뮬레이션과 캐릭터 성장을 중심으로 사용자의 진로 탐색 과정을 이어가는 통합 진로 성장 플랫폼
>
>
> ❝ 막막한 진로 고민을, 하나의 성장 과정으로 ❞
>
> **흩어진 진로 정보를 찾는 데서 끝나지 않고, 직접 선택하고 기록하며 성장하는 서비스입니다.**

---

## ►프로젝트 개요

진로를 탐색하는 과정은 생각보다 복잡합니다.  
사용자는 관심 있는 분야를 찾아보고, 관련 정보를 검색하고, 주변 사람들의 경험을 참고하며 자신의 방향을 정해야 합니다.  
하지만 실제로는 다음과 같은 어려움이 자주 발생합니다.

- 진로 정보, 취업 정보, 대외활동 정보가 여러 곳에 흩어져 있어 필요한 정보를 찾기 어려움
- 관심 분야가 있어도 실제로 어떤 선택을 해야 할지 구체적으로 판단하기 어려움
- 진로 고민을 해도 그 과정이 기록되지 않아 이전 선택과 생각을 다시 확인하기 어려움
- 단순 정보 탐색만으로는 지속적인 동기 부여가 부족함
- 커뮤니티, 정보 탐색, 활동 기록, 성장 요소가 분리되어 있어 사용 흐름이 끊김

이러한 문제를 해결하기 위해,

**All in One**은 사용자의 진로 탐색 과정을 하나의 흐름으로 연결하는 **통합 진로 성장 플랫폼**으로 개발되었습니다.

본 서비스는 **AI 기반 진로 시뮬레이션 + 캐릭터 성장 + 진로 정보 + 커뮤니티 + 활동 기록**을 중심으로

사용자가 자신의 진로 고민을 직접 선택하고, 결과를 확인하며, 성장 과정을 지속적으로 쌓아갈 수 있도록 돕습니다.

특히 All in One의 AI 기능은 서비스 전체에 무리하게 적용된 것이 아니라,  
사용자의 선택에 따라 결과를 제시하는 **진로 시뮬레이션 기능**에 집중되어 있습니다.  
이를 통해 사용자는 단순히 정보를 읽는 데서 그치지 않고, 자신의 선택이 어떤 방향으로 이어질 수 있는지 간접적으로 경험할 수 있습니다.

---

## ►프로젝트 목표

- 사용자가 자신의 진로 방향을 쉽고 자연스럽게 탐색할 수 있도록 지원
- 진로 시뮬레이션을 통해 선택에 따른 결과를 간접적으로 경험할 수 있는 환경 제공
- 캐릭터 성장 시스템을 통해 진로 활동에 대한 지속적인 동기 부여
- 진로 정보, 커뮤니티, 활동 기록을 하나의 플랫폼에서 통합 관리
- 사용자의 진로 탐색 과정을 단순 조회가 아닌 **누적되는 성장 경험**으로 전환
- 모바일 환경에서도 편하게 사용할 수 있는 직관적인 진로 탐색 서비스 구현

## ►주요 기능

### 1. AI 기반 진로 시뮬레이션 🤖

- 사용자가 진로와 관련된 상황을 확인하고, 선택지 중 하나를 선택하면 AI가 선택에 따른 결과를 제공합니다.
- 단순한 정보 검색이 아니라, 사용자가 직접 선택하고 결과를 확인하는 **상황 기반 시뮬레이션 구조**로 구성했습니다.
- 사용자는 시뮬레이션 결과를 통해 특정 선택이 어떤 가능성으로 이어질 수 있는지 간접적으로 경험할 수 있습니다.

    **시뮬레이션 흐름**

    → 사용자가 진로 관련 상황 확인

    → 선택지 중 하나를 선택

    → 선택값을 기반으로 AI 응답 요청

    → AI가 선택에 따른 결과와 피드백 생성

    → 사용자는 결과를 바탕으로 자신의 진로 방향을 다시 고민

    - ✅ **사용자 선택 기반 진행** → 수동적인 정보 탐색이 아닌 능동적인 진로 탐색 가능
    - ✅ **AI 응답 활용** → 선택에 따른 다양한 결과와 조언 제공
    - ✅ **진로 고민 구체화** → 막연한 고민을 실제 선택 상황으로 변환

### 2. 캐릭터 성장 시스템 🌱

진로 탐색은 한 번의 검색으로 끝나기보다, 꾸준히 경험하고 기록하며 쌓아가는 과정에 가깝습니다.  
All in One은 이러한 흐름을 시각적으로 보여주기 위해 **캐릭터 성장 시스템**을 도입했습니다.

- 사용자는 출석, 시뮬레이션 참여, 정보 확인 등 다양한 활동을 통해 경험치를 얻습니다.
- 누적된 경험치에 따라 캐릭터가 성장하며, 사용자는 자신의 활동량을 시각적으로 확인할 수 있습니다.
- 캐릭터는 단순한 장식 요소가 아니라, 사용자의 진로 탐색 과정을 상징하는 성장 지표로 활용됩니다.
- 사용자는 캐릭터의 성장 과정을 통해 자신의 진로 활동이 쌓이고 있다는 느낌을 받을 수 있습니다.

> 사용자의 진로 탐색이 쌓일수록 캐릭터도 함께 성장하는 구조입니다.
>

### 3. 진로 정보 제공 📚

- 진로, 취업, 대외활동, 학습 정보 등 사용자가 참고할 수 있는 정보를 제공합니다.
- 정보는 카테고리별로 구분하여 사용자가 필요한 내용을 빠르게 확인할 수 있도록 구성했습니다.
- 사용자는 정보 목록에서 관심 있는 항목을 선택하고, 상세 내용을 통해 필요한 정보를 확인할 수 있습니다.
- Supabase에 저장된 데이터를 기반으로 정보 목록과 상세 내용을 관리합니다.

### 4. 커뮤니티 기능 💬

- 사용자는 진로 고민, 취업 정보, 스터디 모집, 프로젝트 팀원 모집 등 다양한 게시글을 작성할 수 있습니다.
- 다른 사용자의 글을 확인하며 진로 탐색 과정에서 필요한 의견과 정보를 주고받을 수 있습니다.
- 정보 제공형 서비스에서 끝나지 않고, 사용자 간 경험 공유가 가능한 구조를 제공합니다.
- 커뮤니티 기능을 통해 사용자는 개인적인 진로 고민을 다른 사용자와 연결할 수 있습니다.

### 5. 출석 및 활동 기록 관리 ✅

- 사용자의 출석 여부와 활동 기록을 저장합니다.
- 출석, 시뮬레이션 참여, 정보 확인 등의 활동은 캐릭터 성장과 연결될 수 있습니다.
- 사용자는 자신의 활동 내역을 통해 서비스 이용 흐름과 성장 상태를 확인할 수 있습니다.
- 단순 접속 기록을 넘어, 서비스 재방문과 지속 사용을 유도하는 역할을 합니다.

### 6. 사용자 프로필 관리

- 사용자 정보와 캐릭터 상태를 프로필 화면에서 확인할 수 있습니다.
- 경험치, 레벨, 활동 내역 등을 통해 사용자의 성장 상태를 보여줍니다.
- 개인별 활동 데이터를 기반으로 사용자의 진로 탐색 흐름을 정리합니다.

# 🎨 FE(Frontend)

## ►주요 기능

### 1️⃣ 메인 화면 및 서비스 진입

- **홈 화면 구성**
    - 서비스의 핵심 기능을 한눈에 확인할 수 있도록 구성
    - 진로 시뮬레이션, 정보 페이지, 커뮤니티, 캐릭터 성장 화면으로 이동 가능
    - 사용자가 서비스의 목적과 기능을 빠르게 이해할 수 있도록 주요 기능을 시각적으로 배치
- **시작 화면 제공**
    - 사용자가 서비스를 처음 이용할 때 자연스럽게 진입할 수 있는 흐름 제공
    - All in One의 서비스 목적과 핵심 기능을 간단히 전달
    - 로그인 또는 회원가입 이후 주요 기능으로 이어지는 구조 형성

---

### 2️⃣ 진로 시뮬레이션 화면

- **선택 기반 인터랙션**
    - 사용자가 주어진 상황에서 선택지를 고를 수 있도록 구성
    - 선택 결과에 따라 AI 응답을 화면에 출력
    - 사용자는 복잡한 입력 없이 선택 중심으로 시뮬레이션 진행 가능
- **AI 응답 표시**
    - Gemini API를 통해 생성된 시뮬레이션 결과를 사용자에게 제공
    - 결과, 조언, 다음 행동 방향 등을 확인할 수 있도록 구성
    - 사용자가 자신의 선택을 다시 생각해볼 수 있도록 피드백 형태로 출력
- **사용자 경험 중심 화면 흐름**
    - 질문 → 선택 → 결과 확인의 단순한 구조로 진입 장벽을 낮춤
    - 진로 고민을 어렵지 않게 시작할 수 있도록 화면 흐름을 설계

---

### 3️⃣ 진로 정보 페이지

- **카테고리별 정보 제공**
    - 진로, 취업, 대외활동, 학습 등 다양한 정보 카테고리 표시
    - 사용자가 관심 있는 정보를 빠르게 찾을 수 있도록 구성
- **정보 목록 및 상세 보기**
    - 제목, 카테고리, 본문 내용 등을 상세 화면에서 확인
    - Supabase에 저장된 데이터를 기반으로 정보 목록과 상세 내용을 렌더링
- **정보 탐색 UX**
    - 카드 또는 리스트 형태로 정보를 정리하여 사용자가 쉽게 탐색 가능
    - 진로 관련 정보를 단순 나열하지 않고, 사용자가 필요한 항목을 선택해 확인할 수 있도록 구성

---

### 4️⃣ 커뮤니티 화면

- **게시글 목록 표시**
    - 사용자들이 작성한 게시글을 목록 형태로 확인
    - 진로 고민, 정보 공유, 스터디 모집, 프로젝트 팀원 모집 등 다양한 글 작성 가능
- **게시글 작성 및 조회**
    - 사용자가 직접 게시글을 작성할 수 있는 화면 제공
    - 게시글 상세 내용을 확인할 수 있도록 구성
- **사용자 소통 기반 구조**
    - 정보 제공형 서비스에 커뮤니티 요소를 더해 사용자 간 상호작용 지원
    - 진로 탐색 과정에서 생기는 질문과 경험을 공유할 수 있는 공간 제공

---

### 5️⃣ 캐릭터 및 프로필 화면

- **캐릭터 상태 표시**
    - 사용자의 활동에 따라 성장하는 캐릭터를 화면에 표시
    - 경험치, 레벨, 성장 상태 등을 시각적으로 제공
- **프로필 정보 관리**
    - 사용자 정보와 활동 기록 확인
    - 서비스 이용 현황을 개인 단위로 정리
- **성장 동기 부여**
    - 진로 탐색 활동이 캐릭터 성장으로 이어지도록 구성
    - 사용자가 자신의 활동이 누적되고 있다는 점을 직관적으로 확인 가능

---

### 6️⃣ 출석 화면

- **출석 체크 기능**
    - 사용자가 서비스에 접속하거나 출석 버튼을 통해 출석 기록을 남길 수 있음
- **활동 보상 연결**
    - 출석 기록을 경험치 또는 캐릭터 성장과 연결 가능
- **지속 사용 유도**
    - 매일 접속하고 활동할 이유를 제공하여 서비스 재방문을 유도

---

### 7️⃣ UI/UX 디자인

## ►주요 기능

### 1️⃣ 사용자 인증 및 계정 관리

- 회원가입 시 사용자 정보를 입력 받아 계정을 생성합니다.
- 로그인 시 입력된 정보를 검증하여 서비스 이용이 가능하도록 처리합니다.
- 사용자 정보는 데이터베이스와 연동되어 프로필, 활동 기록, 캐릭터 성장 정보와 연결됩니다.

---

### 2️⃣ 진로 시뮬레이션 API

- 사용자의 선택 값을 서버로 전달하고, AI 응답 생성에 필요한 형태로 가공합니다.
- Gemini API와 연동하여 선택 기반 진로 시뮬레이션 결과를 생성합니다.
- 생성된 응답은 프론트엔드로 반환되어 시뮬레이션 화면에 표시됩니다.
- AI 기능은 전체 서비스에 무분별하게 적용하지 않고, 핵심 기능인 진로 시뮬레이션에 집중하여 사용합니다.

---

### 3️⃣ 진로 정보 관리

- Supabase에 저장된 진로 정보 데이터를 조회하여 프론트엔드에 제공합니다.
- 정보 카테고리, 제목, 본문, 작성일 등의 데이터를 관리합니다.
- 사용자는 프론트엔드에서 정보 목록과 상세 내용을 확인할 수 있습니다.

---

### 4️⃣ 커뮤니티 게시글 처리

- 게시글 작성, 조회, 수정, 삭제 기능을 처리합니다.
- 게시글 데이터는 데이터베이스에 저장되며, 커뮤니티 화면에서 목록 형태로 제공됩니다.
- 사용자의 진로 고민, 정보 공유, 모집 글 등을 저장하고 관리합니다.

---

### 5️⃣ 캐릭터 성장 및 활동 기록 처리

- 출석, 시뮬레이션 참여, 정보 확인 등 사용자의 활동 기록을 저장합니다.
- 활동에 따라 경험치가 누적되고, 누적 경험치에 따라 캐릭터 레벨이 변경될 수 있습니다.
- 사용자의 성장 상태를 프로필 및 캐릭터 화면에서 확인할 수 있도록 데이터를 제공합니다.

---

## ►확장 가능성 및 마무리

All in One은 단순히 진로 정보를 보여주는 서비스에 그치지 않고,

**진로 탐색, 선택 기반 시뮬레이션, 사용자 활동 기록, 캐릭터 성장, 커뮤니티 소통**을 하나의 흐름으로 연결하는 서비스입니다.

- AI 기반 진로 시뮬레이션
- 활동 기반 캐릭터 성장
- 진로 정보 통합 제공
- 사용자 커뮤니티를 통한 경험 공유
- 개인별 활동 기록 기반 맞춤형 성장 구조

이러한 구조는 **진로 탐색을 일회성 검색이 아니라 지속적인 성장 과정으로 전환**하는 데 의미가 있으며,

향후 더 많은 진로 데이터와 사용자 활동 데이터를 연동한다면 **개인 맞춤형 진로 성장 플랫폼**으로 확장할 수 있습니다.
